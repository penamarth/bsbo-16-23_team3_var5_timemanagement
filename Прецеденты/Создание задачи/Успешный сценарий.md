# Сценарий использования: Создание задачи

## Успешный сценарий

1. **Руководитель** инициирует `TaskManagement.CreateTask(managerId, title, description)` и передает в контроллер бизнес-данные задачи.
2. **TaskManagement** извлекает своего создателя через `IUserStorage.GetById(managerId)` и фиксирует, какой пользователь отвечает за создание.
3. **TaskManagement** создает объект `Task` с новой последовательностью идентификаторов и передает себя как `ITaskStorage`, чтобы задача умела сохранять изменения только через контроллер.
4. **Task** автоматически переходит в состояние `NewTaskState`, что фиксирует статус "Создана" без обращения к хранилищу напрямую.
5. **TaskManagement** вызывает роль руководителя (`User.PerformAction(new CreateTaskAction())`), чтобы зафиксировать бизнес-операцию, и возвращает пользователю идентификатор созданной задачи.

## Результат
- В контроллере появляется новая запись `Task`, доступная для дальнейших операций.
- Состояние задачи синхронизировано через `ITaskState`, а сам объект знает, кто является автором.
- Руководитель получает идентификатор задачи и может переходить к назначению исполнителей.
