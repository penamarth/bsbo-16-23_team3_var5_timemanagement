# Сценарий использования: Отслеживание статуса задачи

## Успешный сценарий

1. **Сотрудник** открывает экран задачи и нажимает «Проверить статус», что приводит к вызову `TaskManagement.TrackTaskStatus(employeeId, taskId)`.
2. **TaskManagement** достает профиль исполнителя через `IUserStorage.GetById(employeeId)` и удостоверяется, что запрос исходит от реального пользователя.
3. **Контроллер** извлекает нужную задачу из собственного массива через `ITaskStorage.GetById(taskId)` и получает ссылку на объект `Task`.
4. **Task** запрашивает у своего `ITaskState` человекочитаемое название состояния методом `task.trackStatus()` без обращения к базе.
5. **TaskManagement** логирует результат, связывает его с контекстом пользователя и возвращает сотруднику текст статуса.

## Результат
- Исполнитель видит актуальное состояние задачи (например, «В работе (InProgress)») вместе с идентификатором.
- Контроллер задокументировал обращение, что позволяет менеджеру отслеживать журнал запросов.
- Task по-прежнему не содержит ссылок на базу данных и общается только через `ITaskStorage`.
