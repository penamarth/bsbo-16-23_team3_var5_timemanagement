# Сценарий использования: Учет выполнения задач

## Успешный сценарий

1. **Руководитель** инициирует команду `TaskManagement.TrackTaskExecution(managerId)` из пользовательского интерфейса.
2. **TaskManagement** достает карточку менеджера через `IUserStorage.GetById(managerId)` и удостоверяется в правах.
3. **Контроллер** проходит по своему внутреннему массиву задач и формирует сводку (идентификатор, состояние `Task.trackStatus()`, накопленная выработка).
4. **TaskManagement** активирует `TrackTaskExecutionAction` роли менеджера, чтобы зафиксировать бизнес-действие.
5. **Контроллер** генерирует объект отчета через `ReportGenerator.generateExecutionReport(tasksSummary)` без обращения к базе.
6. **TaskManagement** дополнительно отправляет агрегированную выработку в бухгалтерию методом `sendPayrollData`.
7. **Менеджер** получает результат вызова и просматривает сформированный отчет на экране.

## Результат
- В TaskManagement остается аудит операции и свежая статистика по каждой задаче.
- Руководитель видит визуальный отчет и может принимать решения по загрузке команды.
- Бухгалтерия получает фактические данные по трудозатратам без прямого доступа к задачам.
