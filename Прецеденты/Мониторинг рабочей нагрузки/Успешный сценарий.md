# Сценарий использования: Мониторинг рабочей нагрузки

## Успешный сценарий

1. **Руководитель** нажимает кнопку «Мониторинг нагрузки», инициируя `TaskManagement.MonitorWorkload(managerId)`.
2. **Контроллер** обращается к `IUserStorage.GetById(managerId)`, чтобы удостовериться в полномочиях пользователя.
3. **TaskManagement** пробегает по внутреннему списку задач и агрегирует показатели загрузки (`Task.Workload`, текущие состояния).
4. **Контроллер** запускает `MonitorWorkloadAction` роли менеджера, фиксируя бизнес-операцию.
5. **TaskManagement** формирует текстовый отчет с суммарной нагрузкой по задачам и отображает его руководителю.

## Результат
- Руководитель получает сводку по текущему количеству задач и их трудозатратам.
- TaskManagement сохраняет журнал обращения и может повторно использовать собранные данные для отчетности.
- Процедура выполнена без прямого обращения задач к базе данных, все операции проходят через контроллер и интерфейсы хранилищ.
